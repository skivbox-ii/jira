# Группировка по пользователю и задаче в календаре

## Проблема

В `unified-calendar.js` каждая строка jira-блока и repo-блока дублирует имя пользователя и ключ задачи. При 10+ записях за день это тратит место и затрудняет чтение.

## Решение

Группировка на уровне рендеринга (подход A). Данные (`dayData`) не меняются.

## Порядок группировки

Пользователь → Задача → записи (worklogs, changes, comments, repoItems).

- Если выбран 1 пользователь — заголовок пользователя не показываем, только группировку по задачам.
- Пользователи сортируются по убыванию суммарных часов за день.
- Задачи внутри пользователя — по убыванию часов.
- Записи внутри задачи — по времени.

## Затрагиваемый файл

`ujg-user-activity-modules/unified-calendar.js`

## Структура данных группировки

```
groupDayItems(dayData, issueMap, selectedUsers) → [
  {
    author: { displayName, name },
    showHeader: boolean,
    totalHours: number,
    issues: [
      {
        issueKey, issueSummary, issueStatus, issueStatusChangedAt,
        totalHours,
        worklogs: [],
        changes: [],
        comments: [],
        repoItems: []
      }
    ]
  }
]
```

```
│  └───────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────┘
```

## Новый рендерер

`renderGroupedBlock(groups, issueMap)` заменяет вызовы `renderJiraBlock` + `renderRepoBlock` в `buildCalendarInnerHtml`.

## CSS-классы

- `ujg-ua-user-group` — блок пользователя
- `ujg-ua-issue-group` — блок задачи с левым бордером
