# Timesheet Report Column — Design

## Контекст

Виджет Timesheet (`ujg-timesheet.js`) показывает worklogs по дням в виде календаря. Данные уже загружены в `state.calendarData` и `state.users`. Нужно добавить аналитическую колонку "Отчёт" справа от календаря — сводка по каждому пользователю.

## Решение

Галочка "Отчёт" в панели управления. Когда включена — справа от календаря появляется компактная таблица с метриками по каждому пользователю за выбранный период.

## Layout

```
┌────────────────────────────────────┬────────────────────────┐
│         Календарь (flex-grow)      │    Отчёт (~320px)      │
│                                    │                        │
│  Пн  Вт  Ср  Чт  Пт              │  Пользователь  Списано │
│  ...                               │  Иванов        168h ✓  │
│                                    │  Петров        152h -16│
│  Всего: 488h                       │  Сидоров        —      │
└────────────────────────────────────┴────────────────────────┘
```

- `display: flex` обёртка: календарь слева (flex-grow), отчёт справа (width: 320px)
- На мобильных (< 768px): отчёт переходит под календарь
- Независим от галочки "По разработчикам" — оба можно включить одновременно

## Метрики

Для каждого пользователя:

| Метрика | Вычисление | Формат |
|---------|-----------|--------|
| Списано | Сумма timeSpentSeconds за все дни | `168h`, `42h 30m` |
| Норма | Рабочие дни (Пн-Пт) в периоде * 8ч | Скрытая, для расчёта дефицита |
| Дефицит | Норма - Списано (если > 0) | `-16h` красным |
| Дни | Рабочие дни с worklogs / всего рабочих | `19/21` |
| Задач | Уникальные issue keys | `14` |

## Цветовая индикация

- **Зелёный** (`#27ae60`): списано >= норма
- **Красный** (`#e74c3c`): дефицит > 0 (отображается число дефицита)
- **Серый/прочерк**: нет записей вообще (`—`)
- Без записей — пустая строка, ничего не пишем (как просил пользователь)

## Таблица

- Заголовок: Пользователь | Списано | Дефицит | Дни | Задач
- Сортировка: по умолчанию по имени, клик на заголовок переключает сортировку
- Строка "Итого" внизу: суммы по всем видимым пользователям
- Учитывает фильтр пользователей (state.selectedUsers)

## Данные

Никаких дополнительных API-вызовов. Данные берутся из:
- `state.calendarData` — worklogs по дням
- `state.users` — маппинг userId → displayName
- `state.days` — массив Date объектов в диапазоне
- `state.selectedUsers` — фильтр пользователей

Функция `computeUserReport(userId)`:
1. Считает рабочие дни в `state.days` (Пн-Пт, день недели через `utils.getDayOfWeek`)
2. Для каждого дня фильтрует worklogs по userId из `state.calendarData`
3. Агрегирует: totalSeconds, daysWithEntries, уникальные task keys
4. expectedSeconds = workDays * 8 * 3600
5. deficit = expectedSeconds - totalSeconds (если > 0)
6. Возвращает `{ totalSeconds, expectedSeconds, deficit, daysWorked, workDays, taskCount }`

## Рендер

- `renderReport()` вызывается из `renderCalendar()` если `state.showReport === true`
- Генерирует `<div class="ujg-report-panel">` рядом с календарём
- Обновляется при каждой перерисовке (в т.ч. прогрессивная загрузка — цифры обновляются в реальном времени)

## Норматив

8 часов в рабочий день, фиксированное значение. Выходные (Сб, Вс) не считаются рабочими. Праздники не учитываются.
